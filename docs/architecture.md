# 開発要件: Obsidian Copilot CLI 連携プラグイン

## 概要
Obsidianの右ペイン（サイドバー）にチャットUIを構築し、ユーザーの指示に基づいてローカルにインストール済みの `github copilot cli` (または `@github/copilot-sdk`) をバックグラウンドで呼び出し、Vault内のファイル編集やコンテキスト参照を行うカスタムプラグインを開発する。

copilot sdk
https://github.com/github/copilot-sdk

## 前提条件
- **ターゲット環境:** Obsidian デスクトップ版（Electron / Node.js APIアクセス可能）。
- **外部依存:** ユーザーのローカル環境（PC）には既に `npm install -g @github/copilot` 等によりCLIがインストールされ、`copilot --login` による認証が完了しているものとする。
- **プラグインの役割:** プラグイン自体にCLIを内包するのではなく、Node.jsの `child_process` を介してローカルのCLIプロセスと通信を行う。

## 主要コンポーネントと設計方針

### 1. `main.ts` (プラグインのエントリーポイント)
- Obsidian APIを使用し、右ペイン用のカスタムビュー（`ItemView`）を登録・初期化する。
- Vaultの絶対パスを取得し、後述のサービス層に渡す（`app.vault.adapter` を利用）。

### 2. `CopilotChatView.ts` (UI層)
- Obsidianの `ItemView` を継承し、右ペインにドッキング可能なチャットUIを構築する。
- 技術スタックは、Obsidianプラグイン開発と相性の良い Svelte または React、あるいは Vanilla DOM を使用する（実装しやすいものを選択すること）。
- **機能:**
  - ユーザーからのプロンプト入力エリア。
  - チャット履歴の表示（ストリーミング応答の逐次描画に対応すること）。
  - ローディング（処理中）状態の表示。

### 3. `CopilotService.ts` (制御・実行層)
- Node.jsの `child_process` (`spawn` または `exec`) を使用して、バックグラウンドでCopilot CLIを呼び出す。
- **コンテキストの注入:** 実行時のカレントディレクトリ (`cwd`) には、必ずVaultの絶対パスを指定し、CopilotがVault配下をコンテキストとして認識できるようにする。
- **入出力の処理:**
  - ユーザーのプロンプトをCLIの標準入力に渡し、標準出力（stdout）をリッスンしてUI層にストリーミングで返す。
  - CLIが自律的にVault内のファイルを編集（書き換え）した場合、Obsidian標準のファイル監視機能（File Watcher）によりエディタへ自動反映されるため、プラグイン側での独自同期処理は実装しない。
- **安全性とバージョン管理への配慮:** チーム開発等の共有Vaultで利用されることを想定し、自動編集による破壊的変更のセーフティネットはGit等の外部バージョン管理に委ねる。ただし、CLIからのファイル書き込み操作が行われた際は、UI上にどのファイルが変更されたかのログを明示すること。

## 実装済みの機能 (Version 1.0)
現在、以下の機能がプラグインとして実装済みであり、初期動作要件を満たしています。
1. **右ペインチャット UI (`CopilotChatView`)**: 
   - 履歴表示、ストリーミング描画、ローディング状態のハンドリング。
   - Obsidianからテキストをコピペできる選択可能なチャット履歴。
2. **CLI・SDK 連携 (`CopilotService`)**:
   - `child_process.spawn` を用いた直接プロセス呼び出しによる通信。
   - `stdout` による回答ストリーミングと、`stderr`（遠隔メトリクスやデバッグなど）の分離によるノイズの削減。
3. **パスのユーザー設定 (`settings.ts`)**:
   - GUI（Electron）の制約を回避するため、`copilot` および `node` の絶対パスを環境設定から指定できるように対応済み。
4. **ファイル操作権限 (`--allow-all`)**:
   - Copilotに対してファイルの読み書きをユーザーへの対話プロンプトなしで自律的に実行させるフラグを適用。
5. **コンテキスト・セッション維持 (`--resume`) & 履歴永続化**:
   - チャットごとに固有のUUID (`sessionId`) を維持することで、会話の文脈を引き継ぐ仕様を実装。
   - `data.json` を用いて、Obsidian再起動後も直前のチャット履歴とアクティブなセッションIDを完全に復元する。
6. **Markdown レンダリング (`MarkdownRenderer`)**:
   - レスポンス完了時に生のテキスト表示から Obsidian 標準の Markdown ノードへ置換し、コードブロックやリストをリッチに表示。
7. **アクティブファイルのコンテキスト自動付与**:
   - エディタで現在アクティブなファイルを検知し、ユーザー任意でプロンプトの不可視コンテキストとしてCLIにパスを連携。
8. **ファイル変更イベント可視化 (`vault.on`)**:
   - CLI実行中に発生したバックグラウンドのファイル作成・更新イベントを検知し、チャット画面上に明示的なシステムメッセージとして装飾表示する。
